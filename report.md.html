This is important since in a point cloud the orientation of observation by an autonomous agent could be any direction, #### Implementation: The algorithm for this is quite simple, a python module calls the relevant rotation_x , rotation_y and rotation_z functions as per the command line arguments passed in the evaluation: For example- ``` python eval_seg.py --RotationXYZ 150.0 150.0 150.0 --load_checkpoint best_model ``` The above command would call all three rotation functions and rotate the point cloud equally around x, y and z axis with the angle values written, ```python angle_rad = torch.deg2rad(torch.tensor(angle, dtype=torch.float32)).to(args.device) rotation_matrix_x = torch.tensor([ [1, 0, 0], [0, torch.cos(angle_rad), -torch.sin(angle_rad)], [0, torch.sin(angle_rad), torch.cos(angle_rad)] ], dtype=torch.float32).to(args.device) # Rotation around x axis in rotate_x function rotation_matrix_y = torch.tensor([ [torch.cos(angle_rad), 0, torch.sin(angle_rad)], [0, 1, 0], [-torch.sin(angle_rad), 0, torch.cos(angle_rad)] ], dtype=torch.float32).to(args.device) # Rotation around y axis in rotate_y function rotation_matrix = torch.tensor([ [torch.cos(angle_rad), -torch.sin(angle_rad), 0], [torch.sin(angle_rad), torch.cos(angle_rad), 0], [0, 0, 1] ], dtype=torch.float32).to(args.device) # Rotation around z axis in rotate_z function rotated_tensor = torch.matmul(input_tensor, rotation_matrix) #Here the rotation matrix #depends on the number of chosen rotations ``` #### Results ##### Rotation along X-axis: Classification Results The below are the results of rotation around x-axis note the objects below were correctly classified inspite of the rotation, for comparative analysis the same objects were used which were rotated around x,y and z axis all and their results have some failure cases.
